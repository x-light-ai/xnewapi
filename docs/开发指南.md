## 启动后端

在项目根目录执行：

```bash
# 默认监听 `3000` 端口
go run .

#或者编译exe
go build -o app.exe .
.\app.exe
```

## 启动前端

在 `web/` 目录执行：

```bash
cd web
bun run dev
```

默认访问地址通常是：

```text
http://localhost:5173
```

## 和上游合并

当前仓库关系：

- `origin`：你自己的 GitHub Fork，`https://github.com/x-light-ai/xnewapi.git`
- `upstream`：官方上游仓库，`https://github.com/QuantumNous/new-api`

推荐分支职责：

- `main`：Fork 的同步分支，只用于承接上游更新，不直接开发功能
- `dev`：日常开发分支，用来承接你自己的集成功能
- `feat/*`：功能分支，从 `dev` 拉出，完成后再合并回 `dev`

之后每次和上游同步，推荐这样做：

先在 GitHub 上对 fork 仓库执行 `Sync fork`，把远端 `origin/main` 同步到上游最新状态；然后在本地执行下面这些命令：

```bash
# 1. 拉取 fork 上最新的远端引用
git fetch origin --prune

# 2. 切到 main，并更新本地 main 到 origin/main
git checkout main
git pull origin main

# 3. 回到 dev，把最新 main 合并进 dev
git checkout dev
git merge main

# 4. 把更新后的 dev 推到自己的 fork
git push origin dev
```

日常开发建议：

```bash
# 从 dev 拉功能分支
git checkout dev
git pull origin dev
git checkout -b feat/your-feature

# 开发完成后合并回 dev
git checkout dev
git merge feat/your-feature
git push origin dev
```

注意：

- 不要直接在 `main` 上做功能开发。
- 和上游同步时，按 `fetch upstream -> main 合并 upstream/main -> dev 合并 main` 的顺序执行，方便后续排查冲突。
- 在执行 `checkout`、`merge` 之前，如果当前工作区还有未提交改动，先提交或暂存，避免切分支时卡住。
